# Security Policy

Barndoor takes the security of this software seriously. Thank you for helping
keep the Barndoor Terraform provider and its users safe.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Report a vulnerability privately in one of these ways:

- **GitHub private advisory (preferred):** open
  [**Report a vulnerability**](https://github.com/barndoor-ai/terraform-provider-barndoor/security/advisories/new)
  under this repository's **Security** tab.
- **Email:** [security@barndoor.ai](mailto:security@barndoor.ai).

Please include:

- the provider version (or commit) affected,
- a description of the issue and its impact,
- steps to reproduce — configuration, commands, and any logs, with secrets
  (client secrets, access keys, tokens) redacted,
- any suggested remediation, if you have one.

## What to expect

- We aim to acknowledge your report within 3 business days.
- We will keep you informed of progress toward a fix and coordinate disclosure
  timing with you.
- With your permission, we are happy to credit you once a fix is released.

## Scope

This policy covers the code in this repository (the Terraform provider). For
vulnerabilities in the Barndoor platform or other Barndoor products, see
Barndoor's security contact at <https://barndoor.ai>.
